# تعليمات النشر واستمرارية البيانات (Persistence)

لضمان حفظ التعديلات التي تقوم بها في لوحة التحكم بشكل دائم وعدم ضياعها عند توقف المشروع، يجب اتباع الخطوات التالية:

## 1. قاعدة البيانات (Database)
المشروع مهيأ لاستخدام **PostgreSQL** عبر **Drizzle ORM**. لضمان استمرارية البيانات:
*   يجب توفير قاعدة بيانات PostgreSQL (يمكنك استخدام خدمات مثل [Supabase](https://supabase.com/) أو [Neon](https://neon.tech/) للحصول على قاعدة بيانات مجانية).
*   قم بضبط متغير البيئة `DATABASE_URL` في منصة النشر (مثل Vercel أو Railway) ليكون رابط الاتصال بقاعدة البيانات الخاصة بك.

## 2. تهيئة الجداول (Migrations)
عند ربط قاعدة بيانات جديدة، يجب إنشاء الجداول المطلوبة. يمكنك القيام بذلك بتشغيل الأمر التالي في بيئة التطوير الخاصة بك بعد ضبط `DATABASE_URL`:
```bash
pnpm --filter "@workspace/db" run push
```
هذا الأمر سيقوم بإنشاء جدول `resume_data` والجداول الأخرى تلقائياً.

## 3. متغيرات البيئة المطلوبة (Environment Variables)
تأكد من ضبط المتغيرات التالية في منصة النشر:
*   `DATABASE_URL`: رابط الاتصال بقاعدة البيانات (مثال: `postgres://user:pass@host:port/db`).
*   `PORT`: المنفذ الذي سيعمل عليه الـ API (عادة 8080).
*   `BASE_PATH`: المسار الأساسي (استخدم `/` للنشر على النطاق الرئيسي).

## 4. لوحة التحكم (Admin Panel)
*   كلمة المرور الافتراضية هي: `Zoom100*`
*   المسار للوصول للوحة التحكم هو: `/admin`

بمجرد ضبط هذه الإعدادات، سيتم حفظ أي تعديل تقوم به في قاعدة البيانات بشكل دائم، ولن يتأثر بتوقف الخادم أو إعادة تشغيله.
